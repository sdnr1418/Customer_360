## 🎯 Project Overview
This project aims to build a **Customer 360° view** by merging two powerful data mining perspectives: **Association Rule Mining** (identifying what products are bought together) and **Customer Segmentation** (identifying who the customers are based on behavior). 

By utilizing the **Brazilian E-Commerce Public Dataset by Olist**, we are exploring how product associations differ across various customer segments to provide targeted business recommendations.

## 🛠️ Tech Stack
* **Language:** Python 3.9+
* **Data Handling:** Pandas, NumPy
* **Algorithms:** Apriori, FP-Growth (MLxtend), K-Means (Scikit-learn)
* **Visualization:** Matplotlib, Seaborn

## 📈 Methodology
The project follows a structured 5-phase pipeline:
1. **Phase 1: Preprocessing & EDA** - Cleaning 100k+ orders and merging 8 relational tables.
2. **Phase 2: Association Rule Mining** - Identifying product relationships using Support, Confidence, and Lift.
3. **Phase 3: Customer Segmentation** - Grouping customers via Recency, Frequency, and Monetary (RFM) metrics.
4. **Phase 4: Integration** - Analyzing segment-specific purchasing patterns (The "Customer 360" view).
5. **Phase 5: Insights** - Final business recommendations and strategic reporting.
