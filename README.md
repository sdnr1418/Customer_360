## 🎯 Project Overview
[cite_start]This project aims to build a **Customer 360° view** by merging two powerful data mining perspectives: **Association Rule Mining** (what they buy) and **Customer Segmentation** (who they are)[cite: 3, 4, 307]. [cite_start]We are utilizing the **Brazilian E-Commerce Public Dataset by Olist** to discover how product associations differ across various customer segments[cite: 64, 315, 340].

## 🛠️ Tech Stack
* [cite_start]**Language:** Python 3.9+ [cite: 274]
* [cite_start]**Data Handling:** Pandas, NumPy [cite: 275]
* [cite_start]**Mining Algorithms:** Apriori, FP-Growth (MLxtend) [cite: 47, 277, 328]
* [cite_start]**Clustering:** K-Means with RFM Analysis (Scikit-learn) [cite: 54, 276, 333]
* [cite_start]**Visualization:** Matplotlib, Seaborn [cite: 278, 279]

## 📈 Methodology
[cite_start]The project follows a 4-phase pipeline[cite: 320]:
1. [cite_start]**Phase 1: Preprocessing & EDA** - Cleaning 100k+ orders and merging 8 tables[cite: 67, 70, 318, 321].
2. [cite_start]**Phase 2: Association Rule Mining** - Identifying product relationships using Support, Confidence, and Lift [cite: 122-128, 326].
3. [cite_start]**Phase 3: Customer Segmentation** - Grouping customers via Recency, Frequency, and Monetary (RFM) metrics [cite: 144-153, 331].
4. [cite_start]**Phase 4: Integration** - Analyzing segment-specific purchasing patterns[cite: 183, 335].
